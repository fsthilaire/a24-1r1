Structure de travail

1. Type de stockage​

1.1. Stockage local​

Les fichiers qui sont enregistrés sur un stockage local sont accessibles uniquement lorsque l'unité de stockage est connectée à l'ordinateur.

Les unités de stockage peuvent être le disque interne, une clé USB, un disque externe.

Avantages

Ces fichiers sont accessibles en tout temps. Il n'est pas nécessaire d'avoir un accès à internet.

L'accès est plus rapide, car il n'y a aucun téléchargement.

Inconvénients

Par contre, ces fichiers sont accessibles uniquement par l'ordinateur qui utilise l'unité de stockage local.

Si le fichier est détruit sur l'unité de stockage local, le fichier est généralement perdu et non récupérable. Il est donc important de gérer les copies de sauvegardes correctement.

1.2. Stockage en ligne​

Pour les fichiers qui utilisent un stockage en ligne, il est possible d’y accéder à partir de plusieurs appareils. Par contre, s'il y aucun accès réseau, les fichiers ne sont pas disponibles. Le propriétaire du fichier a le plein contrôle sur la durée de la diffusion et de l’emplacement.

Il existe plusieurs services de stockage en ligne, comme Microsoft OneDrive, Google Drive et DropBox.

Avec ces services, il est possible d'avoir une synchronisation active avec l'unité de stockage local. Lorsqu'un fichier est ajouté dans le dossier local du service, il se met à jour automatiquement lorsqu'il y a une connexion internet.

Par défaut, pour économiser de l'espace disque local, ces services gardent une copie locale des fichiers les plus récents. Il s'occupe de supprimer automatiquement la version locale du fichier. L'ordinateur voit uniquement l'arborescence des fichiers. Les fichiers sont téléchargés à la demande.

Voici un exemple de l'état des fichiers.

  • Le nuage indique que le fichier est en ligne uniquement. (Fichier #1)

  • Le crochet vert avec fond blanc indique que le fichier est actuellement dans l'ordinateur temporairement. Il sera éventuellement effacé. (Fichier #2)

  • Le crochet blanc avec fond vert indique que le fichier aura toujours une copie local. (Fichier #3)

Avantages

Le principal avantage des fichiers en ligne est qu'il existe toujours une copie à l'extérieur de l'ordinateur. En cas de bris de l'ordinateur, il existe toujours une copie du fichier.

Les services de stockage en ligne conservent également quelques versions du fichier. Il est possible de récupérer des fichiers modifiés ou supprimés.

Inconvénients

L'inconvénient principal de cette gestion est que si l'ordinateur n'a pas accès à internet, il n'est pas possible d'accéder aux fichiers.

Également, pour les fichiers volumineux, il faut attendre le téléchargement complet de ceux-ci pour être en mesure de l'utiliser.

Finalement, le dossier est activement surveillé par le service. S'il le dossier a beaucoup de changements, il y aura énormément de synchronisations.

Il n'est pas recommandé d'utiliser une unité de stockage en ligne pour vos projets de programmation.

2. Projets​

2.1. Programmation​

Pour les projets de programmation, il n'est pas recommandé d'utiliser OneDrive et Git.

Git est un outil qui permet de conserver le code du projet sur un serveur et de suivre les modifications. Cet outil vous sera enseigné dans le cours Outils de développement.

Il est préférable d'utiliser un dossier local pour vos projets de programmation. La synchronisation avec les dossiers de stockage en ligne peut causer des problèmes pour les projets de programmation.

attention

Ne créez pas vos projets sous le dossiers Documents de Windows. Selon la configuration de OneDrive, il est possible que ce dossier soit synchronisé.

Voici une structure de dossiers.

C:\
|--- Projets\
	|--- A24\
		|--- 1N1\
			|-Exercices
			|--- TP1\
 			|--- TP2\
		|--- 1W1\
	|--- H25\
		|--- 2N1\
		|--- 2N2\
		|--- 2W1\
attention

Pour vos projets qui n'utilisent pas Git, il est important de penser à une méthode de sauvegarde en cas de problème.

Créez une archive de votre dossier à la fin d'une séance de travail. Copiez cette archive dans votre structure OneDrive.

Il sera possible de récupérer votre travail en cas de problème.

2.2. Scolaire classique​

Les projets scolaires classiques sont généralement des travaux écrits.

Il est préférable de mettre ces travaux dans le dossier OneDrive qui est fourni par le Cégep.

L'utilisation de OneDrive sera présentée plus en détail dans le cours Bureautique pour l'informatique.

Voici une structure de dossiers.

OneDrive\
|---A24\
	|--- Anglais_1\
	|--- Bureautique\
	|--- Francais_1\
	|--- Math\
	|--- 1R1\
|---H25\
	|--- Anglais_2\
	|--- Francais_2\
	|--- Philo_1\

Vos travaux scolaires auront automatiquement.
